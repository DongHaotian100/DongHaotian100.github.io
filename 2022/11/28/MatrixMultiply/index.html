Thread

线程是什么？

操作系统进行进程调度时的最小单位，包含在进程之中，是进程运行中的实际运行单位。一条线程对应一个进程中的一条单一顺序控制流，一个进程可以并发执行多个线程，让每条线程执行不同的任务

多线程

可以理解为多个控制流，多线程之间的通信是比较简单的，因为多个线程之间共享内存

POSIX 线程（POSIX threads）

Portable Operating System Interface，是 IEEE 为要在各种 UNIX操作系统 上运行软件，而定义 API 的一系列互相关联的标准的总称，其正式称呼为 IEEE Std 1003，而国际标准名称为 ISO/IEC 9945。此标准源于一个大约开始于 1985 年的项目。

posix 对应的实际上是一种标准，定义了创建和操作线程的 api，特点为：

  • 共享内存
  • 是一种独立于语言而存在的并行执行的模型

线程的状态信息

  • 就绪：线程在等待可以使用的处理器
    • 等待的原因常有：等 IO、等锁、等触发条件
  • 运行：线程在执行
  • 阻塞：线程正在等待某个时间的发生
  • 终止：线程从开始函数中返回或者调用 pthread_exit

线程取消与同步

  • 线程取消，一个线程可以请求终止同一个进程中的其他线程，但是被取消的线程可以延后执行该取消请求，并且执行这个取消请求时也是可以执行一些特定的程序清理操作。pthread_cancel 可以对线程进行异步终止、延迟终止。其中异步终止可以在任何时间发生，但是延迟终止只能在特定时间。
  • 线程同步，可以控制程序流并访问共享数据，从而并发地执行多个线程。常见 4 种同步模型：
    • 互斥锁
    • 读写锁
    • 条件变量
    • 信号

使用线程的好处

  • 提高程序的响应
  • 有效使用多处理器
  • 改进程序结构，很多程序可以构造为多个、半个工作的线程，并不是非得用一个线程执行
  • 减小占用的系统资源

Pthread 线程库基本语法

pthread 实际上定义了一组 C 语言编程模型、函数、常量，是用一个 pthread.h 头文件和一个线程库实现的，API 中大概有 100 个函数调用，全部是用 pthread 开头的，并且可以分为 4 类：

  • 线程管理，创建、加入线程
  • 互斥体
  • 条件变量
  • 使用互斥锁的线程间的同步管理
