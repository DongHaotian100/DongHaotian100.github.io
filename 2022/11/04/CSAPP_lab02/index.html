Lab02 Bomblab

1.Tools

  • gdb调试工具

    • 使用gdb对bomb可执行文件进行反汇编，并且可以动态调试程序，包括打断点、查看栈帧内容、查看寄存器内容等

    • 常用指令：

      • gdb filename：开始进入gdb对可执行文件进行调试

      • disas：对于当前所处的函数进行反汇编，disas [Function] 指定要反汇编的函数。 如果指定，反汇编命令将产生整个函数的反汇编输出。 disas [Address] 指定要反汇编的地址。

      • break *[address]：在某个地址打断点，记得加星号，因为数字一般默认是行号

      • finish ：命令会执行函数到正常退出该函数

      • testi：单步执行一条机器指令，后缀i表示在机器指令层面，类似的有：

      • x/ <n/f/u> <address>：其中n是一个正整数，表示显示内存的长度，也就是说从当前地址向后显示几个地址（units）的内容，f表示显示的格式（format），address表示的是地址，其中format有以下内容：

      • 符号 含义
        x （hexadecimal）按十六进制格式显示变量
        d （signed decimal）按十进制格式显示变量
        u （unsigned decimal）按十进制格式显示无符号整型
        o （octal）按八进制格式显示变量
        t （binary）按二进制格式显示变量
        a （address）按十六进制格式显示地址，并显示距离前继符号的偏移量(offset)。常用于定位未知地址(变量)
        c （character）按字符格式显示变量
        f （floating）按浮点数格式显示变量

  • objdump反汇编工具

    • 使用objdump可以将整个bomb文件进行反汇编，得到的汇编语言文本文件可以输出到单独的文件，用于宏观上的把握，快速看到每个炸弹所对应的函数内容

    • objdump -d bomb > bomb.s可以将bomb的结果反汇编到bomb.s的文件中

2.BombLab

IA-32体系和X86-64位体系中对于寄存器的使用是有区别的，包括栈指针的使用，所以看汇编主要还是把握一些共性的东西吧，注重整体的理解而不是陷入太小的细节

2.1 Phase01

主要步骤如下：

  • 首先使用gdb的disas或者objdump工具进行反汇编找到phase_1函数的汇编指令

  • 分析汇编代码：

    • 预备知识01：栈里面有很多个栈帧，每个栈帧一般维护着一个函数调用

      • ESP：栈指针寄存器(extended stack pointer，64位下为rsp，为方便讲解，后面不再区分)，其内存放着一个指针，该指针永远指向系统栈最上面一个栈帧的栈顶

      • EBP：基址指针寄存器(extended base pointer， 64位下为rbp)，其内存放着一个指针，该指针永远指向系统栈最上面一个栈帧的底部

      • 一般来说，我们将 %rbp 到 %rsp 之间区域当做栈帧，每调用一个函数，就会生成一个新的栈帧。在函数调用过程中，我们将调用函数的函数称为“调用者(caller)”，将被调用的函数称为“被调用者(callee)”。一般想操作栈帧中的内容就会借助ebp和esp。

      • caller需要知道在哪里获取callee返回的值；

      • caller需要知道传入的参数在哪里，返回的地址在哪里。同时需要保证在“被调用者”返回后，%ebp, %esp 等寄存器的值应该和调用前一致。因此，需要使用栈来保存这些数据。

    • 预备知识02：栈里面有很多寄存器，IA-32中通用的有8个寄存器

      • eax、ecx、edx由caller保存

      • ebx、esi、edi由callee保存

      • ebp由callee保存

        • R[ebp]指向调用者的ebp的值

        • R[ebp] + 4存放返回地址

        • R[ebp] + 8存放第一个入口参数

      • esp是动态变化的

    • 第2行将栈顶指针向下移动，调整栈指针esp的位置，然后将esp的内容也赋值给ebp，我们从main函数进入了phase_1函数的内部，肯定要将入口记下来以便完成递归调用的返回，所以会把calller的esp旧值压入栈中保存，并让新的ebp指向该复制的旧值，以便调用完callee后让ebp返回caller的原本栈底，此时esp也要向下移动8字节空间，而内存单元大小是1个字节，故栈指针向后移动8，即-8。后面如果有callee要保存的寄存器、非局部静态变量时，esp会进一步下移。以下为过程调用时栈的变化示意图：

    • 第3行将$0x402400这个立即数放到esi中，这个数据既然是立即数，那么必然是函数自己提前准备好的参数，后面看到test进行比较时就会反应过来是将我们输入的内容和这里的内容进行比较，此时esi中已经装好了字符串内容

    • 第4行调用了函数strings_not_equal，把phase_1的结果存到eax寄存器中。

    • 第5行使用test，是将两个操作数进行and操作，等价于判断eax是不是0

    • 第6行je会根据test的结果是否为0进行跳转，是0则跳转

    • 第7行的内容仅在je未跳转的情况下执行，会调用bomb函数告诉拆炸弹失败，炸弹爆炸

    • 第8行代表函数执行完了，将esp+8，回收栈空间

    • ret弹出返回地址，将其送给eip寄存器也就是PC，esp返回到caller的栈顶

  • 拆除炸弹：根据上面的分析，phase1主要就是判断输入的字符串是否和提前设置的字符串相等，只需在程序运行时找到函数设定值所处的寄存器并查看其内容即可，于是我们使用x/s 0x402400可以得到字符串的值为Border relations with Canada have never been better.

2.2 Phase02

  • 常见的寄存器的作用：

    • x86_64位条件下，rax一般用于存放要返回的值，rdi、rsi、rdx、rcx、r8、r9分别用于存放当前函数的六个参数值，这七个寄存器均是由caller保存。

  • 分析汇编代码：

    • 2-3行主要用来调整栈帧，保存需要存储的内容

    • 第4行给rsp下移了40个单元的空间，应该是callee有局部变量要存储

    • 第5行将栈指针作为第2个参数，因为rsi一般存放函数的第二个参数

    • 第6行调用一个函数，读入6个数字

    • 第7行将rsp内容和1进行比较

    • 第8行根据7的比较结果是否位1，是就跳转到phase_2+52的位置，否则顺序执行到第9行爆炸

    • 第10行表示无条件跳转

    • 第11行表示将rbx-4的地址中的值赋值给eax，12行将eax的值乘2

    • 第13行将eax和rbx的内容进行比较，相等则会在14行进行跳转，否则在15行爆炸

    • 16行为14行跳转的目标，实现了将rbx中内容+4

    • 17行将rbx中的内容和rbp栈底指针内容进行比较可以判断是否比较已经结束

    • 18行根据17行的反馈，不相等代表没有比较完，要继续跳转回11行进行比较

    • 如此循环，直到比较完后，会进入19行，无条件跳转到23行开始回收栈帧

  • 分析以上结果可以发现，函数要求输入六个数字，并会对这些数字进行验证，验证的方式应当是将前一个数字乘以2不断得到后一个数字，直到输入6个数字就结束，所以正确答案应该是输入1，2，4，8，16，32

2.3 Phase03

  • 分析汇编代码：

    • 直接看第7行进行了一个输入，后面应当是对其值进行一个比较

    • 第8行表示对输入的参数的个数进行判断，如果大于1个参数后面才会跳转，其实sscanf要求的就是2个参数，我们可以单步执行到sscanf看一下，可以发现是要输入两个数字的

    • 第11行表示输入的数字如果比7大会跳转到后面爆炸，所以输入的数字应该是介于1到7之间

    • lea传入两个参数，但是rsp移动了24个单位共4字节，去掉2个参数和上面1到7之间的数字，还得再输入一个数字，所以scanf应该输入的是两个数字

    • 14行的间接寻址jmpq *0x402470(,%rax,8)会使得结果根据输入的第一个数字进行跳转到rax*8+0x402470的内存保存的地址上，可以想办法把0x402470地址开始往后的8个内存单元内容拿出来看一下，发现是一些地址内容：

    • 结果是从这个寄存器中指示的地址开始，是有连续的8个地址的，根据rax中的数字进行计算并跳转，跳转之后会讲eax的值变成某个提前设置好的值，33行再和rsp+12地址中的值，也就是我们输入的第二个数字（即phase_3收到的第二个参数）进行比较，相等的话函数就会跳转到36行回收栈帧顺利退出。

  • 根据上面的分析，我们如果第一个数字输入0，那么跳转的地址应该是0*8+0x402470，对应0x402470地址单元中的第一个地址0x400f7c <phase_3+57>，对应15行和0xcf进行比较，使用p 0xcf可以将其十进制值打出来对应十进制是207，所以我们把第二个数字写成207，发现结果通过

2.4 Phase04

  • 分析phase_4汇编代码：

    • 2行是常见的准备栈帧空间

    • 3-4行表示给rcx和rdx传入两个值，而rcx和rdx一般用于存放函数的第3、4个参数，所以这里应该是我们需要输入两个参数，且通过进入sscanf函数或者打印出0x4025cf的内容可以印证应该是输入两个整数

    • 第6行则是将eax值置为0，用于存储后面scanff读入的参数的个数

    • 第7行输入参数

    • 第8行将eax中的数和2做比较，不是2就会跳转到爆炸函数

    • 10-11行将输入的第一个参数和0xe进行相比，小于14才不会跳转到爆炸函数

    • 13-16行准备了一些参数，然后调用了func4函数

    • 17行根据func4的返回结果，eax为0则爆炸

    • 19行又将phase4的第二个参数和0比较，不是0就顺序执行到后面的爆炸函数

    • 综上，在不分析func4的内容时，可以得知第二个数字为0，且第一个数字<14

  • 分析func4代码：

    • 在执行func4之前，保存了edx中值为14，esi中的值为0，edi中的值为phase的第一个参数

    • 从2-7行看，func_4是传入了两个参数，它们来自edx、esi，用ecx存下edx-esi的值，并且将ecx的值进行逻辑右移31位后再加上edx

    • 8行再将eax（值为edx的值，0xce，对应十进制14）进行算数右移一位得到7

    • 第9行将ecx中的内容变为rax + rsi * 1 = 7

    • 第10行用edi（也就是phase4输入的第一个参数）和ecx的7进行比较，如果小于等于7，11行会跳转到16行，17行会再进行一次比较，如果大于等于7会跳转到22行，函数结束，返回的eax值为0，jne就不会执行跳转到bomb函数。所以可以使得edi=7，也就是第一个参数为7

  • 输入7 0作为两个参数，检验运行结果，果然结果正确，至此第4个炸弹拆解成功：
