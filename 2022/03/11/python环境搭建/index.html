Python环境管理

Created2022-03-11|Updated2023-01-28|Tools
|Post View:

更新或者下载包

配环境时下载包主要有两种方式：conda 和 pip，一般推荐 conda，但是也有一些包只能用 pip 才能搜到。注意安装指定版本的包时，conda 用=指定版本号，pip 用==指定版本号

更新与下载

  • 查看已经安装的包列表：pip list

  • 查看需要升级的包：pip list --outdated

  • 更新包： pip install --upgrade library_name

  • 进入自己的虚拟模块时要用 activate env_name

  • pip install -r requirements.txt 可以自己把需要安装的包都加进去批量安装，例如：

使用指定源下载

比如：pip install tensorflow==2.1 -i https://pypi.tuna.tsinghua.edu.cn/simple

从 -i 部分直接加在日常指令之后即可,如果更换过源，即可不必每次都指定源路径

Conda 的常用指令

包管理

  • 升级全部库： conda upgrade --all
  • 升级一个包 conda update packagename
  • 安装包：conda install packagename
  • 也可以以空格分开包名安装多个包： conda installl numpy pandas scipy
  • 安装固定版本的包：conda install numpy=1.10
  • 移除一个包：conda remove packagename
  • 查看所有包：conda list
  • 删除当前虚拟环境下的某个包：conda remove --name pkg_name
  • 删除某个虚拟环境下的某个包：conda remove --name $env_name pkg_name

管理环境

  • 创建虚拟环境：conda create -n env_name list of packagenaem
    例如：conda create -n env_name pandas
  • 指定 python 版本：conda create -n env_name python2 = 2.7 pandas
  • 激活环境： activate env_name
  • 退出环境 : conda deactivate
  • 删除虚拟环境：conda env remove -n env_name
  • 显示所有虚拟环境：conda env list
  • conda 创建的虚拟环境是在 anaconda 安装目录下的 evens 下，所以使用 pycharm，只要在特定项目配置运行环境就可以了
  • 关闭打开终端时默认进入 base 环境：conda config --set auto_activate_base false
  • 查询已安装环境 conda info --envs

其它指令

有问题，首先试试：conda -h

使用 Jupyter Notebook

准备

  1. 首先在环境下安装 Jupyter Notebook
  2. 激活对应环境
  3. conda install packagename

日常使用

  1. cd 进入到 jupyter 项目文件之中
  2. 激活对应 conda 环境：activate env_name
  3. 打开 jupyter notebook：jupyter notebook
  4. 跳转浏览器，开始编程

安装工具扩展🔧

  1. 首先使用 pip3 install jupyter_contrib_nbextensions 安装扩展包
  2. 在 jupyter 界面安装扩展（在 conda 的安装了 jupyter 的环境下才可以），无权限时可以加 sudo ：jupyter contrib nbextension install

常用扩展工具介绍见：Jupyter扩展插件

配环境的实际记录

安装 d2l 踩坑

在使用 d2l 的包时，出现了 jupyter notebook 内核反复挂掉的情况，故卸载 forge 换成 conda，结果选 3.9 版本就 work 了

使用 conda 新建 d2l 虚拟环境

安装 python

通过本次案例发现，3.8 版本的 python 去安装 d2l 是会出现编译的问题，所以以后出现问题可以从这里解决，或许更高版本的 python 就解决了

安装 d2l, torch, torchvision

在安装等 d2l 包的时候，有两种方法，一种是直接 pip，另一种是先把 d2l 的源码下载下来，cd 到 setup.py 所在的文件夹下，然后使用 python setup.py install

安装 nb_conda

用于切换 jupyter 的内核（3.10 版本可能会下载失败）

环境中包冲突

在一个环境中尽量少放太多包，既然使用 conda 管理不同环境，就有必要按照需求建立多个环境，每个环境满足某一次任务的需求，不要害怕多建环境，旧环境可以删除的
