Docker 使用

首先要说明的是，本方案基于 Docker 部署 Hadoop 集群，所以使用了很多 Docker 使用经验，这里一并总结。

启动 Docker 的两种方式：

  1. 使用 Docker Desktop
  2. 使用指令 systemctl start docker 启动，或者配置开机启动 systemctl enable docker

镜像与容器管理

使用时配合 docker desktop 会更高效

以本次实验中涉及到的运行指令为例：

在构建原始 docker 镜像（用于生成一个 master 和两个 slave 节点）时，使用：

docker run -it -v /home/hadoop/build:/root/build --name ubuntu ubuntu

其中：

  • run 指令表示运行一个镜像，得到一个 container
  • -i 表示开启交互式， -t 表示分配一个tty
  • -v 指令表示将该容器内部的/root/build 文件夹与运行 docker 的本地文件夹/home/hadoop/build 共享，这样比较方便文件的传输
  • --name ubuntu 指令是给启动的容器命名为ubuntu
  • ubuntu 表示启动容器所使用的镜像名称

tty

tty 控制台是一个文本终端，它可以让用户与操作系统进行交互。tty 是一个缩写，代表 “teletype”（电传打字机），因为在早期的计算机时代，用户使用电传打字机与计算机进行交互。现在，tty 控制台通常是通过终端仿真器（如 GNOME 终端或 Konsole）提供的虚拟终端来模拟的。tty 控制台是 Linux 系统的一部分，它允许用户在没有图形界面的情况下使用命令行界面来执行任务。

配置 hadoop 集群

先配置Java，在 hadoop 安装目录，假设为/usr/local/hadoop 目录，修改 JavaHome

Hadoop 配置时的 xml 文件中配置信息是指 Hadoop 集群中各个组件的配置参数，例如 NameNode、DataNode、ResourceManager、NodeManager 等。这些配置信息包括了 Hadoop 集群的基本信息和各个组件的详细配置，如端口号、文件系统路径、内存大小等。在 Hadoop 中，这些参数都是通过 XML 格式的配置文件进行定义和管理，通常包括 core-site.xml、hdfs-site.xml、yarn-site.xml 等。通过修改这些 XML 文件中的配置信息，可以对 Hadoop 集群进行灵活的调整和优化。

core-site.xml

这个文件包含了 Hadoop 的核心配置信息，如文件系统的默认块大小、Hadoop 集群名称节点的地址等。在配置 core-site.xml 文件时，需要设置以下属性：

  • fs.defaultFS：指定默认的文件系统 URL
  • hadoop.tmp.dir：指定 Hadoop 的临时目录
  • io.file.buffer.size：指定文件 I/O 缓冲区大小
  • hadoop.proxyuser.<user>.hosts 和 hadoop. proxyuser.<user>.groups：指定代理用户和他们所属的组

hdfs-site.xml

这个文件包含了 HDFS 的配置信息，如块大小、副本数等。在配置 hdfs-site.xml 文件时，需要设置以下属性：

  • dfs.namenode.name.dir：指定 NameNode 存储元数据的目录。
  • dfs.datanode.data.dir：指定 DataNode 存储块数据的目录。
  • dfs.replication：指定每个数据块应该有多少个副本。
  • dfs.blocksize：指定每个数据块的大小。

yarn-site.xml

这个文件包含了 YARN 的配置信息，如 NodeManager 和 ResourceManager 的地址、内存和 CPU 资源管理等。在配置 yarn-site.xml 文件时，需要设置以下属性：

  • yarn.nodemanager.resource.memory-mb 和 yarn. scheduler. minimum-allocation-mb：指定每个 NodeManager 可用的内存量和最小分配量。
  • yarn.nodemanager.resource.cpu-vcores 和 yarn.scheduler.minimum-allocation-vcores：指定每个 NodeManager 可用的 CPU 数量和最小分配数量。
  • yarn.resourcemanager.hostname 和 yarn.resourcemanager.address：指定 ResourceManager 所在节点的主机名和地址。
