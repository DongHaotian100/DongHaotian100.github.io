Gradle Introduction

Gradle 是一种流行的构建工具，它可以用于构建和管理 Java 项目。下面是一个简单的入门指南，帮助您开始使用 Gradle：

安装 Gradle

首先，您需要在计算机上安装 Gradle。您可以从 Gradle 官方网站下载安装 link

创建 Gradle 项目

现在您可以创建一个新的 Gradle 项目。使用命令行或您喜欢的 IDE，创建一个新的 Java 项目，并在项目根目录中创建一个名为 build.gradle 的文件。这个文件是 Gradle 构建的核心文件。

group 是一个定义项目的唯一标识符，通常以反向域名的形式命名，用于在 Maven 仓库或其他存储库中识别和查找项目。在 Gradle 中，group 属性的值通常用于构建项目的 Maven 坐标 (Maven coordinates)，Maven 坐标由 group、artifact 和 version 三个部分组成。group 的值为 com.example，这是一个惯例，通常是使用公司或组织的域名作为 group 的值，以确保其唯一性。version 的值为 1.0-SNAPSHOT，用于标识项目的版本号。

在构建项目时，Gradle 会使用 group、artifact 和 version 这三个信息，组合成项目的 Maven 坐标，并将其发布到 Maven 仓库或其他存储库中，以供其他人使用和下载。示例代码的依赖项含义是：使用 com.google.guava 作为 group，guava 作为 artifact，30.1-jre 作为 version，从 Maven 仓库中下载并添加到项目的类路径中。

除了从 Maven 仓库中下载依赖项，您也可以通过本地文件系统中的目录或 jar 文件，或者使用本地 Maven 仓库来添加依赖项。这些依赖项通常称为本地依赖项。但是，对于大多数项目来说，从 Maven 仓库中获取依赖项是最常见和最方便的方式。

useJUnitPlatform() 是用于告诉 Gradle 使用 JUnit 5 平台运行测试的方法。在 test 块中使用该方法可以指示 Gradle 使用 JUnit 5 运行测试，这意味着可以使用 JUnit 5 中引入的新特性来编写测试。

在 Gradle 中，JUnit 5 是通过 JUnit Platform Launcher 运行的。因此，在构建文件中使用 useJUnitPlatform() 方法可以让 Gradle 自动为您加载 JUnit 5 平台，并使用其中的测试引擎来运行测试。如果您不使用该方法，则 Gradle 将使用默认的测试引擎来运行测试，这可能不支持 JUnit 5 中的某些特性。

需要注意的是，如果您使用的是 JUnit 4，那么使用 useJUnitPlatform() 方法是没有必要的，因为 JUnit 4 已经有自己的运行器和测试引擎。而对于 JUnit 5，为了使用其中的新特性，您需要在构建文件中指定使用 JUnit 5 平台。

运行 Gradle 任务

现在您可以运行 Gradle 任务来构建项目。您可以使用命令行或您喜欢的 IDE 来运行 Gradle 任务。以下是一些基本任务：

  • gradle build：构建项目
  • gradle test：运行测试
  • gradle clean：清除构建产物

Gradle 还有很多其他任务和插件可用，可以根据您的需要进行配置和使用。

这是一个非常简单的 Gradle 入门指南，但它应该足以帮助您开始使用 Gradle 构建和管理 Java 项目。如果您需要更多的帮助，请查阅 Gradle 官方文档 DOC

Gradle Wrapper

Gradle Wrapper 作用

当您在使用 Gradle 构建您的项目时，Gradle Wrapper 是一个非常有用的工具。它是一个脚本，可以帮助您自动下载并安装指定版本的 Gradle，而无需手动安装 Gradle 本身。

Gradle Wrapper 能够帮助您实现以下目标：

  1. 确保项目的构建使用的是特定版本的 Gradle：如果您的项目依赖于特定版本的 Gradle，Gradle Wrapper 可以确保所有使用该项目的人都使用该版本的 Gradle，以便保持构建的一致性。

  2. 避免手动安装 Gradle：Gradle Wrapper 可以自动下载并安装 Gradle，因此您不必手动下载和安装 Gradle。

  3. 简化团队的开发环境：由于所有人都使用相同的版本的 Gradle，因此您可以确保在团队中的所有人都可以构建您的项目。

Gradle Wrapper 脚本由两个文件组成：gradlew 和 gradlew.bat（Windows 平台上）。这些文件是在项目根目录中创建的，当您运行它们时，Gradle Wrapper 将自动下载并安装指定版本的 Gradle，并使用该版本来构建您的项目。Gradle Wrapper 脚本需要配置文件 gradle/wrapper/gradle-wrapper.properties，该文件指定要下载和使用的 Gradle 版本。

Gradle Wrapper 使用

要使用 Gradle Wrapper，只需在命令行中运行 ./gradlew（Linux 或 macOS）或 gradlew.bat（Windows），Gradle Wrapper 将自动下载并使用指定版本的 Gradle 构建您的项目。

Gradle Wrapper 脚本 gradlew 和 gradlew.bat 是由 Gradle 自动生成的，您不需要编写这些脚本。Gradle 会自动创建这些脚本，以便您可以在不安装 Gradle 的情况下使用 Gradle Wrapper 构建项目。

gradle/wrapper/gradle-wrapper.properties 配置文件指定了 Gradle Wrapper 脚本应该使用的 Gradle 版本。在该文件中，可以指定要使用的 Gradle 版本以及 Gradle 的下载地址。

  2. 如果您的项目包含测试代码，您可以在项目根目录下创建 src/test/java 目录，并在该目录下创建您的测试代码文件。同样，您也需要在 Gradle 构建脚本中指定测试代码文件所在的目录。

在默认情况下，Gradle 会根据项目的构建脚本中的配置，将构建的内容生成到相应的目录中。例如，Java 项目的编译输出目录是 $projectDir/build/classes，测试报告的输出目录是 $projectDir/build/reports/tests。

Setting.gradle

settings.gradle 文件是 Gradle 项目中的一个配置文件，用于配置 Gradle 构建系统的全局设置。这个文件位于项目根目录下，其中包含了一些 Gradle 的全局设置，比如项目名称、项目模块、子项目等。您可以通过编辑 settings.gradle 文件来指定您的项目根目录和子项目目录，以及在构建过程中需要包含的模块。

在一个多模块的 Gradle 项目中，settings.gradle 文件用于指定子项目和它们的路径。子项目可以分别是 Java 应用、库模块、插件模块、web 应用等。您可以在 settings.gradle 文件中定义子项目名称和路径，以便 Gradle 可以找到它们并在构建过程中处理它们。例如：

Gradle Plugin

在 Gradle 中，插件（Plugin）是用于扩展 Gradle 构建系统功能的工具。它们为构建过程提供了许多实用的任务和功能，例如编译代码、打包应用程序、运行测试、生成文档等。您可以通过在构建脚本中应用一个或多个插件来使用它们。每个插件都提供了一些预定义的任务和属性，可以让您更轻松地进行构建和部署操作。

Gradle 提供了许多标准插件，可以在 Gradle 的文档中找到完整的列表。例如，Java 插件（Java Plugin）是一个非常常用的插件，它提供了一些任务和属性来编译和打包 Java 代码。另一个常用的插件是 War 插件（War Plugin），它提供了一些任务和属性来打包和部署 Web 应用程序。

您还可以自己编写插件来扩展 Gradle 构建系统的功能。Gradle 插件是基于 Groovy 或 Kotlin 语言编写的脚本，可以通过 Gradle 的插件机制加载和应用。这个过程涉及到 Gradle 的插件开发 API 和 Gradle 插件发布机制，比较复杂。如果您需要开发自己的插件，可以参考 Gradle 官方文档的相关部分。

在 Gradle 中使用插件很简单，您只需要在您的构建脚本中应用插件，Gradle 会自动下载插件并将其应用到您的项目中。

Gradle 插件存储在 Maven 仓库或 Gradle 插件仓库中，类似于依赖项。当您在构建脚本中应用插件时，Gradle 会自动检查插件是否已经存在于本地缓存中。如果插件不存在，Gradle 将从远程仓库中下载插件并将其缓存到本地缓存中。如果插件已经存在于本地缓存中，Gradle 将直接从缓存中加载插件。
