Computational Graph

Gradient

根据偏导数定义，函数值的变化近似是：

Δf=∂f∂w1Δw1+∂f∂w2Δw2(1)\Delta{f}=\frac{\partial{f}}{\partial{w_1}}\Delta{w_1}+ \frac{\partial{f}}{\partial{w_2}}\Delta{w_2} \tag{1} Δf=∂w1​∂f​Δw1​+∂w2​∂f​Δw2​(1)

位移距离是：

(Δw1)2+Δw2)2(2)\sqrt{({\Delta{w_1}})^2+{\Delta{w_2}})^2} \tag{2} (Δw1​)2+Δw2​)2​(2)

于是函数沿着运动方向的偏导数是：

Δf∣Δw∣=∂f∂w1Δw1+∂f∂w2Δw2(Δw1)2+Δw2)2=(∂f∂w1,∂f∂w2)(Δw1(Δw1)2+Δw2)2Δw1(Δw1)2+Δw2)2)(3)\frac{\Delta{f}}{\mid{\Delta{w}}\mid}= \frac{\frac{\partial{f}}{\partial{w_1}}\Delta{w_1}+ \frac{\partial{f}}{\partial{w_2}}\Delta{w_2}} {\sqrt{({\Delta{w_1}})^2+{\Delta{w_2}})^2}}= (\frac{\partial{f}}{\partial{w_1}}, \frac{\partial{f}}{\partial{w_2}}) \begin{pmatrix} \frac{\Delta{w_1}} {\sqrt{({\Delta{w_1}})^2+{\Delta{w_2}})^2}} \\ \frac{\Delta{w_1}} {\sqrt{({\Delta{w_1}})^2+{\Delta{w_2}})^2}} \end{pmatrix} \tag{3} ∣Δw∣Δf​=(Δw1​)2+Δw2​)2​∂w1​∂f​Δw1​+∂w2​∂f​Δw2​​=(∂w1​∂f​,∂w2​∂f​)​(Δw1​)2+Δw2​)2​Δw1​​(Δw1​)2+Δw2​)2​Δw1​​​​(3)

其中梯度是：

(∂f∂w1,∂f∂w2)(4)(\frac{\partial{f}}{\partial{w_1}}, \frac{\partial{f}}{\partial{w_2}}) \tag{4} (∂w1​∂f​,∂w2​∂f​)(4)

位移向量所指方向的单位向量：

(Δw1(Δw1)2+Δw2)2Δw1(Δw1)2+Δw2)2)(5)\begin{pmatrix} \frac{\Delta{w_1}} {\sqrt{({\Delta{w_1}})^2+{\Delta{w_2}})^2}} \\ \frac{\Delta{w_1}} {\sqrt{({\Delta{w_1}})^2+{\Delta{w_2}})^2}} \end{pmatrix} \tag{5} ​(Δw1​)2+Δw2​)2​Δw1​​(Δw1​)2+Δw2​)2​Δw1​​​​(5)

所以，有以下结论：

  1. 函数在 www 处沿着某方向 Δw\Delta{w}Δw 的方向导数是函数在 www 处的梯度和位移方向的单位向量的内积，也可以说方向导数等于梯度向该方向的投影
  2. 上一条也等价于：方向导数是梯度在该方向的投影
  3. 方向导数是函数沿着各个方向的导数，梯度是一个向量，因此梯度本身有方向
  4. 一个函数在各个方向都有导数，但函数在梯度方向的方向导数是最大的
  5. 函数方向导数的最大值为梯度的模
  6. 垂直于梯度方向的导数为 0

这里有个点要注意，虽然在式子（3）中我们是拿到位移 Δw\Delta{w}Δw 后才推出的沿着运动方向的偏导数以及梯度，但是实际上这个定义并不需要位移 Δw\Delta{w}Δw，所以函数确定了坐标系的一个点之后，在该点的梯度向量是确定的，至于第 5 条，当位移向量和梯度向量重合时，该点积值等于梯度的模，所以此时方向导数取得最大值。

Chain Rule and back Propagation

Vector Function and Jacobi Matrix

可以将一个结点看做一个映射关系，也就是一个函数，对于一个内积结点而言，其输入时一个行向量 www 和一个列向量 xxx，然后该内积映射求得内积结果为一个常数。

计算图中的每个 operator 节点都是多到多（向量到向量，常量可以看做是一维向量，矩阵可以看成多个向量）的映射。

标量函数是什么样的？多个输入值计算出一个值。而向量函数就是输入和输出都是向量的函数，一个 nnn 维向量 www 计算得到 mmm 维向量的映射可以看做是 mmm 个标量函数对这个 nnn 维向量都处理了一次，且处理一次即把该 nnn 维向量的所有维度都处理了：

f(w)=(f1(w)f2(w)⋯fm(w))(6)f(w)= \begin{pmatrix} f_1{(w)} \\ f_2{(w)} \\ \cdots \\ f_m{(w)} \end{pmatrix} \tag{6} f(w)=​f1​(w)f2​(w)⋯fm​(w)​​(6)

对于任意 fi(w)f_i(w)fi​(w) 而言，它显然是个标量函数，输入值为 nnn 维向量，输出为标量

而标量函数我们可以计算它的梯度为：

(∂(f1)(w1)(w)⋯∂(f1)(w1)(w))(7)\begin{pmatrix} \frac{\partial(f_1)}{(w_1)}(w) \cdots \frac{\partial(f_1)}{(w_1)}(w) \end{pmatrix} \tag{7} ((w1​)∂(f1​)​(w)⋯(w1​)∂(f1​)​(w)​)(7)

把所有标量函数都计算梯度就得到一个 mmm 行 nnn 列的矩阵叫做雅可比矩阵：

Jf(w)=(∂(f1)(w1)(w)⋯∂(f1)(w1)(w)⋮⋱⋮∂(f1)(w1)(w)⋯∂(f1)(w1)(w))(8)J_f(w)= \begin{pmatrix} \frac{\partial(f_1)}{(w_1)}(w) & \cdots &\frac{\partial(f_1)}{(w_1)}(w) \\ \vdots & \ddots & \vdots \\ \frac{\partial(f_1)}{(w_1)}(w) & \cdots &\frac{\partial(f_1)}{(w_1)}(w) \end{pmatrix} \tag{8} Jf​(w)=​(w1​)∂(f1​)​(w)⋮(w1​)∂(f1​)​(w)​⋯⋱⋯​(w1​)∂(f1​)​(w)⋮(w1​)∂(f1​)​(w)​​(8)

所以，nnn 维向量映射到 mmm 维向量，则该映射在某个具体点 www 的雅可比矩阵是 m×nm\times{n}m×n 大小的。对于计算图中的每个节点，都可以计算出其对其父节点的雅可比矩阵（父节点不止一个时，每个父节点都可以计算出一个雅可比矩阵），实际上就是求向量函数的梯度嘛，标量函数的梯度是向量，向量函数的梯度是矩阵。

以内积结点为例，输入为 nnn 维向量，输出为标量，所以求得雅可比矩阵为 1×n1\times{n}1×n 的向量，实际上得到的就是梯度向量呀。那在简单一点，输入为 1 维的，输出为标量，那求出来雅可比矩阵就是一个元素，实际上就是导数。

Derivable

一元函数可导和多元函数可导都常见，可导意味着线性近似的误差和自变量之差之比为高阶无穷小。但是多元函数其实也就是标量函数，那向量函数可导是什么情况？答案是，多对多映射在某处可导意味着在该点附近可以被线性映射很好的近似，而该线性近似信息存在于雅可比矩阵中。

Chain Rule

这里只需要知道雅可比矩阵同样适用链式法则即可，复合映射 f(g(h(w)))f(g(h(w)))f(g(h(w))) 在 www 处的雅可比矩阵是 Jf⋅Jg⋅JhJ_{f}\cdot{J_g}\cdot{J_h}Jf​⋅Jg​⋅Jh​，而且复合映射中前一个映射的输出维数等于后一个映射的输入维数，对应前一个雅可比矩阵的列数等于后一个雅可比矩阵的行数

对于下面这样一个计算图而言，计算求损失值的结点（即损失函数这个结点）hhh 对权值向量结点 www 的雅可比矩阵

可以利用链式法则计算结果，先求 hhh 结点对乘法节点 × 的雅可比矩阵，再求乘法节点 × 对 + 节点的雅可比矩阵，再乘上 + 节点对内积 dot product 节点，再乘以内积节点对 www 节点的雅可比矩阵。注意，这里的 hhh 节点对乘法节点 × 的雅可比矩阵是指 loss function 函数对 × 节点处的表达式的整体求雅可比矩阵，不是对一个乘法号求雅可比矩阵，其他的类似。

节点对自己的雅可比矩阵为单位矩阵，这一点可以类比一元函数对自己的导数就是单位元素 1，比如 y=3xy=3xy=3x 对于父节点 xxx 的导数是 3，但是对于 yyy 的导数就是 1。具体在向量函数中怎么理解呢，向量函数对 nnn 元的输入进行 mmm 个 nnn 元映射，但是单个节点对自己的输入就是输出，所以处理完的结果就是个 nnn 元的输出，而每个元素都要求其对全体 nnn 个元素的导数来构成雅可比矩阵，所以只有对自己求导时为 1 ，对其他元素求导都为 0，从而最终得到的是一个 n×nn\times{n}n×n 的单位矩阵，只有对角线全是 1，其余全部为 0。

再以加法节点 + 来看雅可比矩阵的计算，假设加法节点 + 有 nnn 个 mmm 维输入向量（也就是 n 个父节点）V1,V2,V3⋯VnV_1, V_2, V_3 \cdots V_nV1​,V2​,V3​⋯Vn​，每个 VVV 向量有 mmm 个元素 Vn1,Vn2⋯VnmV_{n1}, V_{n2}\cdots V_{nm}Vn1​,Vn2​⋯Vnm​ 那么加法节点的 mmm 个加法运算函数分别为：

f1=V11+V21+V31+⋯+Vn1f2=V12+V22+V32+⋯+Vn2⋯fm=V1m+V2m+V3m+⋯+Vnm(9)\begin{align} f_{1} &= V_{11} + V_{21} + V_{31} + \cdots + V_{n1} \\ f_{2} &= V_{12} + V_{22} + V_{32} + \cdots + V_{n2} \\ & {\cdots} \\ f_{m} &= V_{1m} + V_{2m} + V_{3m} + \cdots + V_{nm} \end{align} \tag{9} f1​f2​fm​​=V11​+V21​+V31​+⋯+Vn1​=V12​+V22​+V32​+⋯+Vn2​⋯=V1m​+V2m​+V3m​+⋯+Vnm​​(9)

可以看到 mmm 个映射函数分别对 nnn 个向量中的 mmm 个元素求导，得到的雅可比矩阵就是一个 mmm 维的单位矩阵，所以 + 节点对其父节点的雅可比矩阵是单位矩阵。

这让我联想到单位矩阵是否可以使用压缩表示，这样在 TinyTorch 这个项目中当遇到结点对自己求雅可比矩阵时就可以实现矩阵压缩，从而提升效率。

Back Propagation

在一次反向传播过程中年，一个节点接收到的内容，实际上就是结果节点（通常是计算损失函数的结点）对自己的雅可比矩阵，然后这个结点再把自己对其父节点的雅可比矩阵向后传递即可。

基于上面举例的计算图，在 ADALINE 模型中的损失函数如下所示：

h(l,x)={−l⋅(w⋅x+b), −l⋅(w⋅x+b)<00, −l⋅(w⋅x+b)≥0(10)h(l,x)= \begin{cases} -l\cdot(w \cdot {x} + b) &, \space-l\cdot(w \cdot {x} + b)\lt0 \\ 0 &,\space -l\cdot(w \cdot {x} + b)\geq{0} \end{cases} \tag{10} h(l,x)={−l⋅(w⋅x+b)0​, −l⋅(w⋅x+b)<0, −l⋅(w⋅x+b)≥0​(10)

举个例子，当 hhh 结点的父节点的值小于 0 时，求得偏导数为 -1 ，所以其传递给父节点的信息是[1, -1]，这两个值分别是 hhh 结点对自己和对父节点的偏导数。把自己对自己的偏导数传递给父节点干什么？这其实是一个特例，因为 hhh 已经没有子节点了，原本一个节点传给父亲节点的东西应该是[子节点对自己的雅可比矩阵，自己对父亲节点的雅可比矩阵]。注意看，这里自己对自己的偏导数就是 1，而实际应用中像 hhh 节点这样的终端节点可以根据其数据维度生成一个单位矩阵作为自己的雅可比矩阵。

继续反向传播， hhh 节点的父节点即 × 节点传递给其父节点 + 节点的东西就是：[hhh 节点传递给 × 节点的两个导数之积，x 节点对 + 节点的导数]，后续传递以此类推

上面这种计算图是比较简单的，但复杂的计算图中一个节点 A 可以有很多个子节点 sss，这个时候怎么办比较合适呢？即将最终结果 hhh 节点对 A 的各个子节点的雅可比矩阵和各个子节点对 A 的雅可比矩阵乘积后求和即可：

JA=∑JhsJsA(11)J_{A}=\sum{J_{hs}}{J_{sA}} \tag{11} JA​=∑Jhs​JsA​(11)

所以我们也可以清晰的看到，计算图中反向传播的过程可以使用递归来实现。

其实对于 forward 的过程而言，也是先调用最终节点（例如损失函数节点）的 forward 函数，然后函数递归调用父节点的 forward 函数，最终真正的计算从递归栈的最上面（对应最初输入的数据节点）开始计算，所以看起来是从计算图父节点向子节点前进的这样一个 forward 过程。而 backward 实际上是从结果节点开始计算前面父节点的雅可比矩阵，但调用时是先调用父节点的 backward 函数再逐层递归调用至最终节点。

在 backward 过程中计算每个计算节点（operator）的雅可比矩阵时，要先看看该节点有没有存储的 Jacobi 矩阵值，如果有了代表已经计算过了，那么就不需要重新计算，提升效率。这主要是考虑某些复杂的计算图中的一次 backward 过程可能会多次计算某个节点。

同时，并不是计算图中的每个节点在 backward 过程中都参与计算，如果一个节点的 value 值不为空，那么说明其参与本次计算，那么可以递归调用该节点的 backward 方法求得最终节点对其的雅可比矩阵。

最后，关于梯度平均的问题，每次 backward 执行后计算到的梯度是平均值。总共数据 x 有 4 组数据，每个 epoch 将四组数据都计算一遍梯度值，再除以 4 得到的是该 epoch 下的反向传播梯度值，用这个平均梯度值去更新参数，这意味着这里使用的是全批量梯度下降法。另外，每个 epoch 中计算的 4 个梯度是累加，而更新时使用减法，毕竟是梯度下降而不是梯度上升，这符合我们最小化 losslossloss 函数的目标。

Summary

After the variables in the computation graph are initialized by init, the forward method is called on the result node to recursively calculate the values of each node on the path. The information propagates forward along the computation graph and finally gets the value of the result node. Afterwards, backward method is called on nodes that need updating to recursively calculate Jacobi matrix of each node on the path with respect to the result node, and information is propagated backwards.

If multiple nodes need to be updated, such as weight vector nodes and bias nodes, call the backward method on each of these nodes respectively. Since the Jacobi matrix values of intermediate nodes have been saved in the Jacobi attribute, it is not necessary to calculate repeatedly when calling its backward method multiple times on multiple nodes. This is the core content of back propagation, which implements the chain rule of differentiation, saves intermediate results, and thus exchanges time for space.

Gradient Descend

基于计算图我们重新审视一下梯度下降的过程。梯度下降即让损失函数的值沿着梯度方向逐渐下降，而损失函数下降的过程就是让输入函数的参数变化的过程，如何让某个参数变化？那就是用损失函数求关于该参数的导数，然后另该参数值减去学习率和该导数的乘积。

在计算图中的过程具体是这样的：

  1. 初始化（调用系统初始化）各节点以及对 value 节点赋值（手动），亦即确定初始参数和喂第一组数据
  2. 调用 forward 方法，计算最终要输出的结果节点的值
  3. 在所有 trainable 的节点上调用 backward 方法
  4. 调用 backward 方法后，变量节点的 Jacobi 矩阵属性保存了结果节点对它的梯度的转秩，用于更新参数
  5. 清楚节点值以及 Jacobi 矩阵值，重新喂新数据，从第 2 步重复执行

并不是每次更新参数都能保证损失函数在任意数据上都朝着函数值下降的方向发展，但通常经过整个 epoch 来看，还是会对降低模型在全体训练样本上的损失值有重要作用。之所以用一个或者部分数据计算出的梯度可以更新到参数上，基于的就是随机梯度下降法 SGD 的原理（一个数据计算梯度对应 SGD 算法，一小批数据计算梯度对应 Minibatch Gradient Descent）。

随机梯度下降法

随机梯度下降法（SGD）是一种迭代优化算法，它是梯度下降法的一种变体。它不是以所有训练数据的平均值来计算梯度，而是以一个随机样本来计算梯度。SGD可以用于解决大型机器学习问题，因为它只使用一小部分的数据来进行迭代优化。此外，由于它只使用一个样本来计算梯度，因此其速度要快得多。SGD还可以在数据集中加入噪声时保持稳定性。

在 SGD 过程中我们还要考虑一个参数随机初始化的问题，深度学习中的参数初始化是指将神经网络的权重和偏置初始化为特定的值。这些初始值对神经网络的训练过程有很大的影响，因此选择合适的参数初始化方法非常重要。

模型的初始参数设置并不是一个个手动赋值，一般来说，可以采用随机初始化、零初始化、Xavier/Glorot 初始化、Kaiming 初始化以及自定义初始化五种方法进行参数初始化。其中，随机初始化是最常用的方法，它将权重随机地赋予一个接近0的小数值；而零则是将所有权重都设为0；Xavier/Glorot 和 Kaiming 则是对随机初始化进行优化；自定义则是根据不同任务选择不同的参数来起作用。

关于梯度的计算方法，实际常用雅可比矩阵，前面已经提到了雅可比矩阵的相关知识，详细的 Jacobi 矩阵数学知识后面会有篇幅单独介绍。
